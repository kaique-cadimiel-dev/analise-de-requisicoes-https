# Contribuindo

Obrigado por contribuir! Siga estes passos para reportar problemas ou enviar PRs.

1. Abra uma issue descrevendo o problema ou a feature desejada.
2. Faça um fork do repositório e crie uma branch com um nome descritivo.

```bash
git checkout -b feat/minha-melhora
```

3. Escreva código claro e mantenha o estilo existente. Testes são bem-vindos.
4. Abra um Pull Request descrevendo as mudanças e os motivos.

## Padrões e dependências
- Use um ambiente virtual (`venv`)
- Atualize `requirements.txt` quando precisar adicionar dependências

## Contato
Se precisar de ajuda rápida, abra uma issue e marque o responsável pelo projeto.
